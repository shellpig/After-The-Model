# 主角公寓

> 第一個可玩場景。此文件只記錄主角公寓場景內的敘事、互動物、驗收方向。

## 場景定位

主角某一天在一間公寓中醒來。

他不知道這裡是哪裡，也不記得自己是誰。當他嘗試開門離開時，發現門被上鎖，而且自己不知道如何打開。

第一個關卡是小型室內探索：

```text
醒來
-> 調查房間
-> 收集奇怪線索
-> 線索逐步組合成身份 / 住處 / 職業資訊
-> 主角意識到這是自己的公寓
-> 找到打開公寓門的方法
-> 離開公寓
```

## 已完成互動

### 床

節點：

```text
BedSleepArea
```

目前行為：

- 靠近顯示：`E: 睡覺去`
- 按 E 顯示訊息：
  ```text
  你心中有事, 睡得並不好...
  ```

後續定位：

- 初期敘事提示。
- 可作為「主角狀態不安」的第一個低風險互動。

### 櫥櫃

節點：

```text
CabinetStorageArea
```

目前行為：

- 靠近顯示：`E: 打開櫥櫃`
- 按 E 打開容器 UI。
- 再按 E 關閉目前 prototype 容器 UI。
- 外觀：暗木 / 暗褐。

目前程式現況：

```text
cabinet_storage = 5x3
```

規格目標：

```text
cabinet_storage = 5x6
```

用途：

- 玩家長期倉庫。
- 後續改為雙欄容器 UI 後，櫥櫃應在右欄，玩家背包在左欄。

### 冰箱

節點：

```text
FridgeStorageArea
```

目前行為：

- 靠近顯示：`E: 打開冰箱`
- 按 E 打開容器 UI。
- 外觀：米白 / 淡藍 / 冷灰。
- 容量：5x2。

用途：

- 小型容器。
- 後續可放食物、便條、線索物。

## 待新增互動

### 大門

節點：

```text
DoorExitArea
```

Script：

```text
interactable_area.gd
```

參數：

```text
interaction_id = "door_exit"
prompt_text = "E: 開門"
message_id = "door_locked"
required_knowledge = "identity_door_unlock_method"
```

`interactable_area.gd` 目前沒有 `required_knowledge` 欄位。實作大門時需新增：

```gdscript
@export var required_knowledge := ""
```

位置：

- 公寓右側門前。
- 互動區由使用者手動微調。

靠近提示：

```text
E: 開門
```

未解鎖時按 E：

```text
門上了鎖, 而你發現自己不知道如何打開...
```

Message id：

```text
door_locked
```

`apartment_room.gd` 的 `MESSAGES` 必須新增：

```gdscript
"door_locked": "門上了鎖, 而你發現自己不知道如何打開...",
"door_opened": "你想起來了。門鎖不是壞了, 是你忘了操作方式。"
```

解鎖條件：

```gdscript
GameState.has_knowledge("identity_door_unlock_method")
```

解鎖後行為：

- MVP 不做場景切換。
- 若已取得知識，按 E 顯示：
  ```text
  你想起來了。門鎖不是壞了, 是你忘了操作方式。
  ```
- 後續接 `SceneRouter` 後，再把成功分支改成轉場離開公寓。

派發規格：

```gdscript
match current_interactable.interaction_id:
  "door_exit":
    if GameState.has_knowledge(current_interactable.required_knowledge):
      show_message("door_opened")
    else:
      show_message("door_locked")
```

驗收：

- 未取得 `identity_door_unlock_method` 時，靠近大門顯示 `E: 開門`。
- 未取得知識時按 E，只顯示 `door_locked` 訊息，不切場景、不移動玩家。
- 取得知識後按 E，顯示 `door_opened` 訊息。
- 大門不檢查背包物品，不依賴鑰匙 item。

## 第一關筆記設計

分類：

```text
身份
工作
線索
```

主角公寓初期應以 `線索` 為主，最後轉成 `身份`。

範例線索：

```text
冰箱門上貼著一串數字。
門旁的感應器有長期使用的刮痕。
桌上散落著 AI 善後相關的文件。
櫥櫃裡有一件看起來熟悉的工具。
```

範例身份筆記：

```text
這裡是我的公寓。
我似乎從事 AI 善後相關工作。
我知道公寓門的開法。
```

## 公寓通關條件

目標：

```text
取得 identity_door_unlock_method
```

取得方式尚未定稿，但方向是：

```text
調查房間中的多個線索
-> 線索被解析 / 組合
-> 新增身份筆記 identity_door_unlock_method
-> 大門可開啟
```

## UI 行為要求

公寓場景應遵循全遊戲通用 UI 規格：

- 世界模式：主角可走路，E 與最近互動物互動。
- 大型 UI 開啟時：主角不能移動。
- Prompt 永遠跟隨主角上方。
- MessageBox 自動適配文字尺寸並逐字顯示。
- 容器互動最終要改成雙欄：
  ```text
  左：背包
  右：容器
  ```
