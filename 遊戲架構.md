# After-The-Model 遊戲架構

> 本文件記錄全遊戲通用系統。場景專屬內容請放在 `subdocs/地點/`。

## 目前狀態

- 引擎：Godot 4.6.3 / GDScript。
- 視角：純 2D 側捲，主角只左右移動。
- 已有可跑原型：`apartment_room.tscn`。
- 已有基礎系統：
  - 玩家左右移動與 walk 動畫。
  - 互動區 `Area2D`。
  - 互動提示框跟隨主角。
  - 共用訊息框，支援逐字顯示。
  - 共用容器 UI，依容器資料切換標題、格數、外觀色。
  - 多個互動區重疊時，選離玩家最近的互動物。

## 輸入設計

所有輸入走 Godot InputMap，不在程式中直接監聽實體按鍵。

| Action | 鍵盤 | 用途 |
|---|---|---|
| `move_left` | A / Left | 世界移動；UI 開啟時改為焦點移動 |
| `move_right` | D / Right | 世界移動；UI 開啟時改為焦點移動 |
| `interact_primary` | E | 世界互動；UI 主要動作 |
| `interact_secondary` | R | UI 查看 / 取消等次要動作 |
| `interact_tertiary` | T | UI 丟棄等第三動作 |
| `open_inventory` | I | 開 / 關背包，待實作 |
| `open_notebook` | J | 開 / 關筆記，待實作 |
| `ui_cancel` | Esc | 關閉目前 UI，待實作 |

## UI Mode

後續需要建立單一 UI 模式狀態，避免角色移動與 UI 操作互相衝突。

```text
NONE
INVENTORY
CONTAINER
NOTEBOOK
MESSAGE
CONFIRM
```

規則：

- `NONE`：玩家可移動，E 與附近物件互動。
- 任何大型 UI 開啟時，玩家不能移動。
- UI 開啟時，WASD / 方向鍵改為移動 UI 焦點。
- `Esc` 關閉目前 UI。

## 互動系統

每個可互動物件使用 `Area2D`，掛 `interactable_area.gd`。

基本欄位：

```gdscript
interaction_id: String
prompt_text: String
message_id: String
```

目前做法：

- `Area2D` 只負責偵測玩家進出。
- 場景 controller 負責：
  - 選目前互動物。
  - 顯示 prompt。
  - 按 E 後執行對應行為。
- 多個互動區重疊時，選離玩家最近的 `CollisionShape2D`。

## Prompt

互動提示使用共用 `PromptPanel`。

目前已完成：

- 有 UI 底圖。
- 文字由互動物或目前 UI 狀態決定。
- 位置跟隨主角，水平大致置中在主角上方。

## MessageBox

共用訊息框用於短文本敘事與無法執行某動作時的回饋。

目前已完成：

- 訊息框依完整文字自動調整大小。
- 文字逐字出現，速度為每秒 3 字。
- 含標點符號一起逐步顯示。

後續規則：

- `MESSAGE` mode 中，`E` 或 `Esc` 關閉訊息。

## 容器系統

容器使用資料驅動。

目前 prototype 使用單一 `ContainerPanel`：

```gdscript
CONTAINERS = {
  "cabinet_storage": {
    "title": "櫥櫃",
    "cols": 5,
    "rows": 3,
    "skin": "cabinet",
    "panel_position": Vector2(...)
  },
  "fridge_storage": {
    "title": "冰箱",
    "cols": 5,
    "rows": 2,
    "skin": "fridge",
    "panel_position": Vector2(...)
  }
}
```

已完成：

- 不同容器可設定 title、cols、rows、skin、panel_position。
- 容器 panel 會依格數自動調整大小。
- `cabinet` skin：暗木 / 暗褐。
- `fridge` skin：米白 / 淡藍 / 冷灰。

規格目標：

- 背包：5x3。
- 櫥櫃：5x6，作為玩家倉庫。
- 冰箱：5x2。
- 目前程式中的櫥櫃仍是 5x3，待調整到 5x6。

## 背包系統

待實作。

背包是玩家自身容器：

- `I` 開 / 關。
- 單獨打開時只顯示背包，位置置中。
- 格數：5x3。
- 顯示 `Credits`，不佔格。
- 全鍵盤 / 手把友善，不依賴滑鼠。

背包操作：

```text
方向鍵 / WASD = 選格
E = 使用 / 裝備 / 卸下
R = 查看
T = 丟棄
Esc / I = 關閉
```

## 容器雙欄操作

待實作。

打開櫥櫃 / 冰箱等外部容器時，UI 改為雙欄：

```text
左：玩家背包
右：外部容器
```

操作：

```text
方向鍵 / WASD = 選格 / 跨欄
E = 移動 1 個到另一側
R = 查看
T = 丟棄
Esc = 關閉
```

規則：

- 可堆疊物品按 E 一次只移動 1 個。
- 目標側若已有同物品且未滿 stack，先補該 stack。
- 補不下時開新 stack。
- 沒有空格時顯示「放不下了。」。
- 背包或容器都會自動整理，同類型物品合併並排序，空格集中在後面。
- 劇情重要物品不可丟棄。

## 筆記 / 知識系統

待實作。

筆記分類：

```text
身份
工作
線索
```

定義：

- `身份`：穩定、已確認的自我資訊，例如這裡是自己的公寓、自己的職業、門鎖開法。
- `工作`：委託、任務、零工、工作紀錄。
- `線索`：暫時、不完整、尚未理解的資訊，可轉化為身份或工作筆記。

操作：

```text
J = 開 / 關筆記
Esc = 關閉
Left / Right = 切分類
Up / Down = 選筆記
選到筆記後自動顯示全文
```

資料範例：

```gdscript
{
  "id": "identity_door_unlock_method",
  "category": "身份",
  "title": "門鎖的開法",
  "body": "你想起來了。門鎖需要先按住門把下方的舊感應片, 再用右手手套靠近讀取器。",
  "status": "active"
}
```

## 知識解鎖

門與部分劇情進展使用知識 id 判斷，不直接讀筆記文字。

範例：

```gdscript
has_knowledge("identity_door_unlock_method")
```

目前規格：

- 大門離開公寓的條件是知識，不是物品。
- 玩家需要從房間探索中取得足夠線索，最後形成穩定知識。
