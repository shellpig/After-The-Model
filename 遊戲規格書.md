# After-The-Model 遊戲規格書

> 本文件記錄全遊戲通用系統的規格與驗收條件。場景專屬規格放在 `subdocs/地點/`，由主規格書 link 過去。
>
> 每個系統段落含「驗收」清單：可逐條 check 的行為條件，用於手動驗收與已知問題追蹤。

## 目前狀態

- 引擎：Godot 4.6.3 / GDScript。
- 視角：純 2D 側捲，主角只左右移動。
- 已有可跑原型：`apartment_room.tscn`。
- 已有基礎系統：
  - 玩家左右移動與 walk 動畫。
  - 互動區 `Area2D`。
  - 互動提示框跟隨主角。
  - 共用訊息框，支援逐字顯示。
  - 共用容器 UI，依容器資料切換標題、格數、外觀色。
  - 多個互動區重疊時，選離玩家最近的互動物。
  - `GameState` / `UIMode` autoload。
  - 背包 UI（`I` 開關、5x3、Credits、item icon、鍵盤 focus）。
  - 筆記 UI（`J` 開關、三分類 tab、列表 + 全文、Page Up / Down 捲動）。
  - 容器雙欄 UI（櫥櫃 5x6 / 冰箱 5x2、左背包右容器、跨欄 focus、E 移動 1 個、stack 自動合併）。
  - FloatingToast（容量不足顯示「放不下了。」，1.5s + 0.5s fade 自毀）。
  - `move_up` / `move_down` InputMap action（WASD 上下方向綁定）。
  - UI 開啟時凍結主角移動，並以半透明 overlay 保留世界可見。

## 輸入設計

正式遊戲行為走 Godot InputMap，不在程式中直接監聽實體按鍵。例外：UI focus 元件可在已啟用 UI mode 時補實體 WASD fallback，作為方向鍵導覽鏡像；不得觸發世界互動或資料變更。

| Action | 鍵盤 | 用途 |
|---|---|---|
| `move_left` | A / Left | 世界移動；UI 開啟時改為焦點移動 |
| `move_right` | D / Right | 世界移動；UI 開啟時改為焦點移動 |
| `interact_primary` | E | 世界互動；UI 主要動作 |
| `interact_secondary` | R | UI 查看 / 取消等次要動作 |
| `interact_tertiary` | T | UI 丟棄等第三動作 |
| `open_inventory` | I | 開 / 關背包，已完成 Phase 1-B |
| `open_notebook` | J | 開 / 關筆記，已完成 Phase 1-C |
| `ui_cancel` | Esc | 關閉目前 UI，背包 / 訊息 / 容器已接入 Phase 1-B |

### 驗收

- 所有會改變遊戲狀態的玩家輸入均經 `InputMap`，程式不得直接用 `KEY_E` / `KEY_R` / `KEY_T` 等實體鍵碼觸發互動或資料變更。
- 任一 action 重新綁定後，所有相依行為（移動、互動、UI 焦點）均跟隨更新。

## UI Mode

單一 UI 模式狀態，避免角色移動與 UI 操作互相衝突。

```text
Autoload name: UIMode
File: res://scripts/autoload/ui_mode.gd
Scope: 跨場景共用
```

模式列舉：

```text
NONE
INVENTORY
CONTAINER
NOTEBOOK
MESSAGE
CONFIRM   # Phase 1-E 啟用，由 ConfirmDialog 使用
```

規則：

- `NONE`：玩家可移動，E 與附近物件互動。
- 任何大型 UI 開啟時，玩家不能移動。
- UI 開啟時，WASD / 方向鍵改為移動 UI 焦點。
- `Esc` 關閉目前 UI，狀態回到 `NONE`。
- 同時只能有一個大型 UI 開啟；要切到另一個 UI 必須先關閉當前 UI。
- 背包 / 筆記 / 容器 UI 開啟時，**世界仍可見**（半透明 overlay 蓋在世界之上，不全黑遮蓋），方便邊看世界邊操作 UI。

最小 API：

```gdscript
signal mode_changed(new_mode: int)

enum Mode { NONE, INVENTORY, CONTAINER, NOTEBOOK, MESSAGE, CONFIRM }

func get_mode() -> int
func set_mode(mode: int) -> void
func is_world_input_blocked() -> bool   # 當前 mode != NONE 時為 true
```

### 大型 UI 之間互切

`I` / `J` 等大型 UI 鍵在當前 mode != NONE 時的行為：

- 當前 mode 與按鍵對應的 UI **相同** → 關閉，回到 `NONE`。
- 當前 mode 與按鍵對應的 UI **不同** → **先關閉當前 UI、再開新的**（一鍵切換，不要求玩家先 Esc）。

例：
- INVENTORY 中按 `J` → 背包關、筆記開、mode = NOTEBOOK。
- NOTEBOOK 中按 `I` → 筆記關、背包開、mode = INVENTORY。
- CONTAINER 中按 `I` → 容器關、背包開、mode = INVENTORY（雙欄容器同規則）。
- MESSAGE 中的大型 UI 鍵 → 同上邏輯（先關訊息、再開目標 UI）。

實作上 `UIMode` 保持純狀態管理，只更新 mode 並 emit `mode_changed(new_mode)`。場景 controller（目前為 `apartment_room.gd`）負責監聽 signal，關閉舊 UI、開啟新 UI、同步 overlay / prompt / focus。

### CONFIRM mode 的 caller 還原

`CONFIRM` mode 由 `ConfirmDialog` 使用，**不是**大型 UI，行為與互切規則不同：

- 進入 `CONFIRM` 時 UIMode 記住 caller mode（如 `INVENTORY` 或 `CONTAINER`），不關閉 caller UI（背包 / 雙欄維持顯示但停止接鍵）。
- 離開 `CONFIRM` 時還原 caller mode，焦點回 caller UI。
- `CONFIRM` 期間 `I` / `J` 等大型 UI 鍵不反應；只接 `E` / `Esc`。

```gdscript
# UIMode 內部
var _caller_mode := Mode.NONE

func enter_confirm() -> void:
    _caller_mode = current_mode
    current_mode = Mode.CONFIRM
    mode_changed.emit(Mode.CONFIRM)

func exit_confirm() -> void:
    current_mode = _caller_mode
    _caller_mode = Mode.NONE
    mode_changed.emit(current_mode)
```

### 驗收

- 任何 UI 開啟時，主角不能左右移動、不會觸發世界互動。
- UI 關閉後恢復 `NONE`，主角可移動且 Prompt 重新出現。
- 同時只能有一個大型 UI 開啟。
- 背包 / 筆記 / 容器 UI 開啟時，背景仍透出世界畫面（透明度由半透明 overlay 控制）。
- `mode_changed` signal 在每次切換時 emit。
- 大型 UI 互切時舊 UI 確實隱藏，無雙重 panel 疊加。
- 進 / 離 `CONFIRM` mode 時 caller UI 不關閉、焦點正確還原。

## 共用 UI 元件

跨 UI 重複使用的元件，集中規範以避免漂移。實作上應抽成獨立 scene / script。

### BagGrid

玩家背包格子的視覺與資料 component。

- 5x3 = 15 slot，sizing / skin / focus 視覺依「背包系統」段。
- 資料源：`GameState.get_inventory()`，listens `inventory_changed` signal。
- 同一個 `BagGrid` 同時用於：
  - 1-B 背包獨立 UI（INVENTORY mode）
  - 1-D 雙欄左欄（CONTAINER mode）
- 接受外部設定當前 focused index 與「是否捕捉鍵盤輸入」flag（決定誰主導鍵盤）。
- 視覺絕對一致；改背包視覺只能改這一處。

### FloatingToast

短暫提示，不切 UI Mode。

```text
位置        = 視觸發 panel 上方 8px，水平置中於該 panel
字          = 18px 米白 (0.94, 0.92, 0.84, 1.0)
背景        = 半透明黑 (0, 0, 0, 0.7) + 8px padding
顯示時間    = 1.5 秒
淡出時間    = 0.5 秒
重複觸發    = 取代既有 toast、計時器重置；不疊加
```

API：

```gdscript
FloatingToast.show(text: String, anchor_node: Control = null)
```

用途清單：
- 1-D「放不下了。」
- 1-E「無法丟棄這個物品」
- 1-E「這類裝備已經滿了，先卸下身上的再裝備新的。」
- Phase 2+ 解謎條件未滿足等臨時提示

### PanelFooterHint

panel 底部一行操作鍵提示。

```text
字          = 14px 暗灰 (0.55, 0.55, 0.55, 1.0)
位置        = panel 底部 padding 內，左對齊
格式        = "<KEY>: <action>    <KEY>: <action>    ..."
```

API：

```gdscript
PanelFooterHint.set_hints(panel: Control, hints: Array)  # [["E", "移動"], ["Esc", "關閉"]]
```

各 UI 對應 hint（依 Phase 進度）：

| UI | Phase 1-B | Phase 1-C | Phase 1-D | Phase 1-E |
|---|---|---|---|---|
| 背包（單獨） | `Esc / I: 關閉` | — | — | `E: 裝備 / 卸下    R: 查看    T: 丟棄    Esc / I: 關閉` |
| 筆記 | — | `Esc / J: 關閉` | — | — |
| 雙欄（兩欄各一，同字串） | — | — | `E: 移動    Esc: 關閉` | `E: 移動    R: 查看    T: 丟棄    Esc: 關閉` |
| ItemDetailModal | — | — | — | `R / Esc: 關閉` |
| ConfirmDialog | — | — | — | `E: 確定    Esc: 取消` |

> 「使用 consumable」MVP 不接（無飢餓 / 血量系統），1-E footer 不寫「使用」字樣；對 `category=consumable` 物品按 E 沉默，無 toast。Phase 3+ 加血量系統後再補。

### ItemDetailModal

R 鍵查看物品 description 用的 modal。**不切 UIMode**（INVENTORY / CONTAINER 不變），開啟時 capture 鍵盤，關閉後焦點回原 grid 格。

```text
panel 寬     = 320
panel 高     = 400
位置         = 蓋在背包 panel 或雙欄左欄上方中央
skin         = inventory
```

內部排版（垂直）：

```text
[ icon 128x128，置中 padding 16 ]
[ name 22px 米白，置中 ]
[ description 16px，換行可讀，左右 padding 16 ]
[ flex spacer ]
[ category tag 14px 暗灰，置中於 panel 底 padding 內 ]
```

`category tag` 規則：

| `category` | tag 文字 |
|---|---|
| `key_item` | `劇情物品（不可丟棄）` |
| `equipment` | `裝備（{equipment_slot 中文}）`，例 `裝備（手持）` |
| `consumable` | `消耗品` |
| `misc` | `物品` |

`equipment_slot` 中文映射：`clothing → 衣服`、`hand → 手持`、`accessory → 其他`。

API：

```gdscript
ItemDetailModal.show(instance_id: String, anchor_node: Control = null) -> void
ItemDetailModal.close() -> void
```

操作：

- `R` 或 `Esc` → 關閉並回到 grid focus。
- 其他鍵：modal 開啟期間全部 block（不讓背包 / 雙欄處理）。

### ConfirmDialog

通用確認 dialog。1-E 用於丟棄確認。切 `UIMode.CONFIRM`，開啟期間 capture 所有鍵盤。

```text
panel 寬     = 360
panel 高     = 160
位置         = viewport 置中
skin         = inventory
```

內部排版：

```text
[ message 18px 米白，置中 padding 16 ]
[ flex spacer ]
[ footer hint: "E: 確定    Esc: 取消" ]
```

API：

```gdscript
ConfirmDialog.show(message: String, on_confirm: Callable, on_cancel: Callable = Callable()) -> void
```

操作：

- `E` → 呼叫 `on_confirm` → 關閉 dialog → UIMode 回先前 mode（INVENTORY / CONTAINER）。
- `Esc` → 呼叫 `on_cancel`（如有）→ 關閉 dialog → UIMode 回先前 mode。
- 其他鍵：block。

> ConfirmDialog 切 `CONFIRM` mode；關閉時必須回到 dialog 開啟**之前**的 mode（INVENTORY 或 CONTAINER），而非預設 NONE。UIMode 內部需記住 caller mode。

## 互動系統

每個可互動物件使用 `Area2D`，掛 `interactable_area.gd`。

基本欄位：

```gdscript
interaction_id: String
prompt_text: String
message_id: String
```

目前做法：

- `Area2D` 只負責偵測玩家進出。
- 場景 controller 負責：
  - 選目前互動物。
  - 顯示 prompt。
  - 按 E 後執行對應行為。
- 多個互動區重疊時，選離玩家最近的 `CollisionShape2D`。

### 驗收

- 玩家站在互動區內時，Prompt 顯示對應 `prompt_text`。
- 玩家離開互動區後 Prompt 立刻消失。
- 同時站在多個互動區內時，只顯示最近一個的 Prompt，按 E 也只觸發該物件。

## Prompt

互動提示使用共用 `PromptPanel`。

目前已完成：

- 有 UI 底圖。
- 文字由互動物或目前 UI 狀態決定。
- 位置跟隨主角，水平大致置中在主角上方。

### 驗收

- 主角移動時 Prompt 跟著移動，不會落後或抖動。
- Prompt 不擋住主角頭頂的關鍵互動範圍。
- 任何 UI 開啟時 Prompt 隱藏，UI 關閉後恢復。

## MessageBox

共用訊息框用於短文本敘事與無法執行某動作時的回饋。

目前已完成：

- 訊息框依完整文字自動調整大小。
- 文字逐字出現，速度為每秒 3 字。
- 含標點符號一起逐步顯示。

後續規則：

- `MESSAGE` mode 中，`E` 或 `Esc` 關閉訊息。

### 驗收

- 訊息框尺寸依完整文字長度自動調整，不出現截字或大量留白。
- 文字以每秒 3 字逐字顯示，含標點。
- `MESSAGE` mode 中主角不能移動，按 E 或 Esc 收掉訊息後恢復 `NONE`。

## 容器系統

容器使用資料驅動。

目前 prototype 使用單一 `ContainerPanel`：

```gdscript
CONTAINERS = {
  "cabinet_storage": {
    "title": "櫥櫃",
    "cols": 5,
    "rows": 3,
    "skin": "cabinet",
    "panel_position": Vector2(...)
  },
  "fridge_storage": {
    "title": "冰箱",
    "cols": 5,
    "rows": 2,
    "skin": "fridge",
    "panel_position": Vector2(...)
  }
}
```

已完成：

- 不同容器可設定 title、cols、rows、skin、panel_position。
- 容器 panel 會依格數自動調整大小。
- `cabinet` skin：暗木 / 暗褐。
- `fridge` skin：米白 / 淡藍 / 冷灰。

規格目標：

- 背包：5x3。
- 櫥櫃：5x6，作為玩家倉庫。
- 冰箱：5x2。
- 目前程式中的櫥櫃仍是 5x3，待調整到 5x6。

### 資料層接 GameState（Phase 1-D）

雙欄打開時容器 panel **必須**讀 `GameState.get_container(container_id)`，不再用空 Button grid。

- 場景在 `_ready()` 呼叫 `GameState.configure_container("cabinet_storage", 30)` / `configure_container("fridge_storage", 10)` 初始化。
- 預載資料寫法見 `subdocs/地點/主角公寓.md > 預載資料`。
- panel 顯示時：foreach slot → 若有對應 inventory entry 顯示 icon + 數量；空格顯示 slot skin 底。
- listen `GameState.container_changed(container_id)` signal 即時刷新。
- 移動物品請呼叫 `GameState.move_one_item_to(...)`，禁止 panel 自行操作 GameState 內部資料。

### 驗收

- 不同容器顯示的 title、格數、skin 完全由 `CONTAINERS` dict 驅動，程式不得 hardcode。
- 容器 panel 寬高隨 cols/rows 自動調整。
- 櫥櫃顯示 5x6，冰箱顯示 5x2，背包顯示 5x3。
- 容器 UI 開啟 / 關閉時遵守 UI Mode 規則（主角不能移動、Prompt 隱藏）。
- 容器 panel 內容由 `GameState.get_container()` 提供；空格顯示 skin 底色，有物品顯示 icon + 數量。
- 容器內容變動時透過 `container_changed` signal 即時刷新，無需重開 panel。

## 背包系統

已完成 Phase 1-B 基礎 UI 與人工驗收。物品操作（E 使用 / 裝備、R 查看、T 丟棄）留 Phase 1-E。

背包是玩家自身容器：

- `I` 開 / 關。
- 單獨打開時只顯示背包，位置置中於 viewport。
- 格數：5x3。
- 顯示 `Credits`，不佔格。
- 全鍵盤 / 手把友善，不依賴滑鼠。
- 開啟時走半透明 overlay，世界仍可見（依 UI Mode 規則）。

背包操作：

```text
方向鍵 / WASD = 選格
E = 使用 / 裝備 / 卸下（Phase 1-E）
R = 查看（Phase 1-E）
T = 丟棄（Phase 1-E）
Esc / I = 關閉
```

Phase 1-B 期間，背包內按 E / R / T 必須被 UI 吃掉但不產生任何效果、log 或提示。

### Panel 尺寸與排版

沿用容器系統常數：

```text
SLOT_SIZE       = 64 x 64
SLOT_GAP        = 4
PANEL_PADDING   = (16, 16)
TITLE_HEIGHT    = 24
```

5x3 計算：

```text
內容寬 = 5 * 64 + 4 * 4 = 336
內容高 = 3 * 64 + 2 * 4 = 200
panel 寬 = 336 + 32 = 368
panel 高 = 200 + 32 + 24 = 256
```

於 1280×800 viewport 置中：左上座標 ≈ `(456, 272)`。

### 標題列（含 Credits）

```text
[ 背包                                    Credits: 0 ]
```

- 左對齊「背包」字樣（18px，inventory skin 標題字色）。
- 右對齊「Credits: <值>」（18px，同色），值來自 `GameState.get_credits()`，聽 `credits_changed` signal 即時更新。
- Credits 內聯於標題列，**不另佔行**。

### Inventory Skin（區別於 cabinet / fridge）

```text
skin id        = "inventory"
panel 底       = (0.12, 0.14, 0.18, 0.88)   # 深藍灰
panel 邊框     = (0.78, 0.42, 0.20, 1.0)    # 主角橙紅
slot 底        = (0.08, 0.10, 0.12, 0.80)
slot 邊框      = (0.30, 0.32, 0.36, 1.0)
標題字色       = (0.94, 0.92, 0.84, 1.0)   # 米白
```

實作建議在 `_apply_container_panel_style` 加 `inventory` 分支（或重構為 dict 對應）。

### 格內顯示

- **Icon**（必有，path 由 `ITEMS_DB[item_id].icon_path` 提供）。
- **數量**：`quantity > 1` 時顯示於右下角，14px，白底深字（或加字陰影確保可讀）。
- **已裝備標記**：右上角 12×12 像素 `E` 字符號，白底深字。1-B 階段預載 item 未裝備，此標記預留位置；1-E 實裝裝備系統後啟用。
- **缺 icon**：顯示紅底白色 `?` placeholder（48×48 居中），方便 dev 期發現未補 icon。

### Focus 視覺（鍵盤導航）

選中格用 2px 亮橙紅外框（同 panel 邊框色 `(0.78, 0.42, 0.20, 1.0)`），蓋在 slot 邊框上，視覺優先順序高於既有 hover。

```text
未選中 / 滑鼠 hover    = 走既有 slot 邊框
鍵盤 focus              = 2px 橙紅外框（dedicated focus_style）
```

Focus 邊界**不繞回**：上 / 下 / 左 / 右抵邊後不移動。

### 半透明 Overlay

開啟背包 / 筆記 / 容器 UI 時，世界上覆蓋一層半透明黑色：

```text
overlay color = (0, 0, 0, 0.3)
```

overlay 在 panel 之下、世界之上。背包 / 筆記 / 容器三者共用同一份 overlay 規格。

- [x] `I` 開啟時 panel 置中於 viewport，尺寸 368×256，左上 ≈ (456, 272)。

### 跨欄 input 規則

無 Tab 鍵（手把不便）。靠方向鍵 / WASD 抵欄邊自動跳對欄：

- 右欄最左行按 ← → 跳左欄
- 跨欄時**保持當前 row index**；對方 row 不足時落對方最後一 row
  - 例：背包 focus 在 row 2（最下），→ 跳 cabinet row 2 或 fridge row 1（fridge 只有 row 0-1）

### Knowledge / Notes API

```gdscript
func has_knowledge(id: String) -> bool
func add_knowledge(note: Dictionary) -> void
func get_notes(category: String) -> Array[Dictionary]
func get_all_notes() -> Array[Dictionary]
```

`add_knowledge(note)` 規則：

- `note.id` 必填。
- `note.category` 必填，合法值為 `身份` / `工作` / `線索`。
- `note.title` 必填。
- `note.body` 必填。
- `note.status` 若未提供，預設為 `active`。
- 同一 `id` 重複新增時不新增第二筆，可更新既有 note 內容。
- 若 `note.category == "身份"`，同時設定 `knowledge[note.id] = true`。
- 若其他系統需要將線索轉成穩定知識，應新增一筆 `身份` note，而不是直接改線索文字。
- `get_notes()` 與 `get_all_notes()` 回傳 deep copy，避免 UI 或外部系統直接修改 GameState 內部資料。

### Inventory API

```gdscript
func get_inventory() -> Array[Dictionary]
func add_item(item_id: String, count: int = 1) -> bool
func remove_item(item_id: String, count: int = 1) -> bool
```

MVP 背包容量固定 15 格。背包資料以 slot array 表示，空格為 `{}`。

Slot schema：

```gdscript
{
  "instance_id": "item_0001",
  "item_id": "old_work_badge",
  "quantity": 1
}
```

規則：

- 可堆疊物品以 `item_id` 合併。
- 移動堆疊物品時，按一次 E 只移動 1 個。
- 先補目標側未滿 stack，再開新 stack。
- 沒空格回傳 `false`，UI 顯示「放不下了。」。
- 每次新增、移除、移動後自動整理，空格集中在後面。
- `add_item(item_id, count <= 0)` 與 `remove_item(item_id, count <= 0)` 立即回傳 `false`，不改狀態、不 emit signal。
- `get_inventory()` 回傳 deep copy，避免 UI 或外部系統直接修改 GameState 內部資料。

### Equipment API

```gdscript
func equip(instance_id: String) -> bool
func unequip(equipment_type: String, slot_index: int) -> bool
func unequip_by_instance(instance_id: String) -> bool     # Phase 1-E helper
func is_equipped(instance_id: String) -> bool             # Phase 1-E helper
func get_equipment() -> Dictionary
```

裝備槽：

```text
clothing: 1
hand: 2
accessory: 2
```

規則：

- 裝備槽滿時 `equip()` 回傳 `false`，UI 顯示：
  ```text
  這類裝備已經滿了，先卸下身上的再裝備新的。
  ```
- 裝備不會離開背包，只是被 equipment 參照。
- 已裝備物品在背包排序中放最前面；卸下後自動整理重排。
- `get_equipment()` 回傳 deep copy，避免 UI 或外部系統直接修改 GameState 內部資料。
- `unequip_by_instance(instance_id)`：當 UI 焦點在背包格時直接用 instance_id 卸下對應裝備，不需先查 slot_index。內部仍走 `unequip` 同樣的整理邏輯。

成功時 emit `equipment_changed`，並 emit `inventory_changed`（因為背包排序變動）。

### External Container API

MVP 中外部容器由 GameState 管理 in-memory runtime state。容器 id 規則：

```text
player_inventory = 保留 id，代表玩家背包
cabinet_storage = 櫥櫃容器，目前 5x6 = 30 slots
fridge_storage = 冰箱容器，目前 5x2 = 10 slots
```

外部容器 id 使用 lowercase underscore，並盡量與 interactable / scene id 一致。空 slot 為 `{}`。

```gdscript
func configure_container(container_id: String, slot_count: int) -> void
func get_container(container_id: String) -> Array[Dictionary]
func move_one_item_to(target_container_id: String, instance_id: String) -> bool

# Phase 1-D 加入；用於場景預載外部容器內容
func seed_container(container_id: String, item_id: String, count: int) -> bool
```

`configure_container()` 規則：

- 若 `container_id` 尚未初始化，建立指定數量的空 slot。
- 若容器已存在，不覆蓋既有內容。

`get_container()` 規則：

- 回傳指定容器的 deep copy。
- 未設定的容器回傳空陣列 `[]`。

`move_one_item_to()` 規則：

- 動態搜尋 `instance_id` 來源，來源可能是玩家背包或任一外部容器。
- MVP 只支援 `player_inventory -> external container` 與 `external container -> player_inventory`。
- 不支援 `container -> container`；若來源是外部容器且目標不是 `player_inventory`，立即回傳 `false`。
- 堆疊物品一次只移動 1 個。
- 先檢查目標是否可容納：有同 `item_id` 且未滿 stack，或有空 slot。
- 若目標無法容納，立即回傳 `false`，來源與目標都不可變動。
- 成功時先扣來源 1 個，再加入目標；來源數量歸零時清空該 slot。
- 新開 slot 時產生新的 `instance_id`。
- 成功後來源與目標都自動排序。
- 只回傳 `true` / `false`，不回傳 UI message、不 log UI message。UI 層收到 `false` 後自行決定顯示文字，例如「放不下了。」。

`seed_container()` 規則（Phase 1-D 新增）：

- 用途：場景預載，把指定數量的某 item 放入外部容器。
- 必須先 `configure_container()` 否則回傳 `false`。
- 行為等同對該容器多次呼叫 `add_item` 的容器版本：補同 id 未滿 stack → 開新 slot；自動產生 `instance_id`；操作後自動整理。
- 容量不足時：能放多少放多少，未能放完仍回傳 `false`；放完則回傳 `true`。
- 成功時 emit `container_changed(container_id)`。
- 僅限場景 `_ready()` 或 dev tool 呼叫；遊戲流程內物品搬運必須走 `move_one_item_to`。

### Discard API（Phase 1-E）

```gdscript
func discard_item(instance_id: String) -> bool
```

通用丟棄入口，由 T 鍵流程呼叫。

規則：

- 動態搜尋 `instance_id` 來源（玩家背包或任一外部容器）。
- 若該 `instance_id` 對應 item 的 `discardable=false` → 立即回傳 `false`，狀態不變。
- 若該 item 已裝備 → 立即回傳 `false`，狀態不變（必須先卸下；UI 端 1-E 把已裝備品的 T 接到「先卸下」流程，或直接擋住）。
- 否則：將 `instance_id` 對應的 1 個（若為 stackable）或整 slot（非 stackable）從來源 slot 移除；數量歸零時清空 slot；操作後來源自動整理。
- 成功時 emit `inventory_changed`（背包來源）或 `container_changed(container_id)`（容器來源）；若該 item 為已裝備品經卸下後再丟棄，額外 emit `equipment_changed`。
- 只回傳 `true` / `false`，不回傳 UI message。UI 層收到 `true` 顯示「已丟棄 {item_name}」toast；收到 `false` 顯示「無法丟棄 {item_name}」toast。

> ⚠️ MVP 階段「stackable 物品按 T 一次丟 1 個」：與 `move_one_item_to` 對稱；不提供整 stack 一次丟。

### Signals

```gdscript
signal inventory_changed
signal container_changed(container_id: String)
signal credits_changed(new_value: int)
signal knowledge_added(id: String)
signal notes_changed
signal equipment_changed
```

Transfer signal 規則：

- `player_inventory -> external container`：emit `inventory_changed`，並 emit `container_changed(target_container_id)`。
- `external container -> player_inventory`：emit `inventory_changed`，並 emit `container_changed(source_container_id)`。
- `count <= 0` 或 atomic 檢查失敗時不 emit。

### 驗收

- `GameState` 以 `GameState.has_knowledge("identity_door_unlock_method")` 可被任何場景直接呼叫。
- 新增同一筆 note 兩次，不會在筆記 UI 中出現重複條目。
- `add_credits(-999)` 或 `set_credits(-1)` 不會讓 credits 小於 0。
- 背包容量超過 15 格時 `add_item()` 回傳 `false`。
- `add_item(item_id, 0)`、`add_item(item_id, -1)`、`remove_item(item_id, 0)`、`remove_item(item_id, -1)` 回傳 `false`，且不改狀態。
- 裝備槽滿時 `equip()` 回傳 `false`，且不替換既有裝備。
- `get_inventory()`、`get_container()`、`get_equipment()` 回傳值被外部修改時，不會改到 GameState 內部資料。
- `configure_container("fridge_storage", 10)` 後可取得 10 格空容器。
- `move_one_item_to("fridge_storage", instance_id)` 可將背包物品移入冰箱；`move_one_item_to("player_inventory", instance_id)` 可將冰箱物品移回背包。
- 嘗試 `container -> container` 回傳 `false`，且不改狀態。
- 目標容器滿格時移動回傳 `false`，來源數量不可減少。
- 背包與容器移動成功時 emit 正確的 `inventory_changed` 與 `container_changed(container_id)`。
- MVP 不寫入磁碟；重開遊戲狀態可重置。

欄位規則：

- `icon_path` 必填，指向 64x64 透明 PNG。
- `name` 顯示於查看 R Modal 與 FloatingToast 訊息。
- `description` 顯示於查看 R Modal 描述區。
- `category` 列舉：`key_item` / `equipment` / `consumable` / `misc`。決定 R Modal 底部 tag 文字與 E 鍵行為。
- `equipment_slot` 空字串代表非裝備品；非空時必須為 `clothing` / `hand` / `accessory` 三者之一。
- `stackable` true 時 stack 規則由 `max_stack` 控制；false 時每個 instance 各占一格。
- `discardable` false 時 T 鍵被 `discard_item` API 攔下；UI 端顯示 FloatingToast「無法丟棄 {name}」。
- `usable` MVP 不接 use 行為（無飢餓 / 血量系統），欄位保留待 Phase 3+。

`Credits` 不放入物品格，而是玩家狀態數字。

## 裝備系統

待實作（Phase 1-E）。

裝備槽：

```text
衣服 (clothing)：1
手持 (hand)    ：2
其他 (accessory)：2
```

僅 `category=equipment` 且 `equipment_slot != ""` 的物品可裝備。

E 鍵行為（**只在 INVENTORY mode、focused 在背包格時生效**；CONTAINER mode 中 E 永遠是「移動」）：

| Focused item | E 行為 |
|---|---|
| 未裝備 equipment | 裝備到對應 slot；槽滿時顯示 FloatingToast「這類裝備已經滿了，先卸下身上的再裝備新的。」 |
| 已裝備 equipment | 卸下 |
| `category=consumable`（usable=true） | **沉默**（MVP 無 use 效果） |
| `category=key_item` 或 `misc` | **沉默** |

排序與視覺：

- 已裝備品在背包排序中固定置頂；同 `item_id` 多裝備時內部順序不強規範。
- 卸下後自動整理重排（依 GameState 自動整理規則），focus 保持 row/col index。
- 已裝備品在背包格右上角顯示 `E` 標記（1-B 已預留位置，1-E 啟用渲染）。

### 驗收

- 衣服槽 1、手持槽 2、其他槽 2，超過時不可裝備。
- 已裝備品在背包排序中固定置頂；卸下後自動重排，focus index 不變。
- 對未裝備品按 E 是「裝備」，對已裝備品按 E 是「卸下」，互不混淆。
- 槽滿時顯示 FloatingToast「這類裝備已經滿了，先卸下身上的再裝備新的。」，原裝備不替換。
- 已裝備品右上角顯示 `E` 標記；卸下後標記消失。
- INVENTORY 中對 `consumable` / `key_item` / `misc` 按 E 完全沉默（無 toast、無動作）。
- CONTAINER 雙欄中 E 永遠是「移動」，不會觸發裝備行為。

## 筆記 / 知識系統

已完成 Phase 1-C 基礎 UI 與人工驗收。

筆記分類：

```text
身份
工作
線索
```

定義：

- `身份`：穩定、已確認的自我資訊，例如這裡是自己的公寓、自己的職業、門鎖開法。
- `工作`：委託、任務、零工、工作紀錄。
- `線索`：暫時、不完整、尚未理解的資訊，可轉化為身份或工作筆記。

操作：

```text
J = 開 / 關筆記
Esc = 關閉
Left / Right = 切分類
Up / Down = 選筆記
選到筆記後自動顯示全文
```

UI 規則：

- 開啟時走半透明 overlay，世界仍可見（依 UI Mode 規則）。
- 三分類顯示為頂部 tab；當前 tab 高亮。
- 同一面板左側顯示該分類筆記列表，右側顯示選中筆記全文（無需另開 Modal）。
- 玩家進入筆記 UI 時預設 focus 在第一個非空分類的第一筆筆記。

### Panel 尺寸與排版

```text
panel 寬 = 880
panel 高 = 560
位置     = viewport 置中（1280x800 viewport 下左上 ≈ (200, 120)）

內部分區：
  TAB_BAR_HEIGHT     = 36         # 頂部三分類 tab
  LIST_PANE_WIDTH    = 240        # 左側列表欄
  PADDING            = 16
  全文欄寬           = panel 寬 - LIST_PANE_WIDTH - PADDING * 3 = ~600
  內容區高           = panel 高 - TAB_BAR_HEIGHT - PADDING * 2 = ~492
```

### Skin

沿用 1-B 已定的 `inventory` skin（深藍灰底 + 主角橙紅邊框 + 米白標題字），與背包同一視覺家族。

### Tab Bar

```text
[ 身份 ]  [ 工作 ]  [ 線索 ]
```

- 每個 tab 固定寬 120px、高 = TAB_BAR_HEIGHT (36px)。
- 文字 18px。
- **未選中** tab：文字暗灰 `(0.55, 0.55, 0.55, 1.0)`，無底線。
- **選中** tab：文字米白 + 底部 2px 橙紅線（同 panel 邊框色 `(0.78, 0.42, 0.20, 1.0)`）。
- 鍵盤 Left / Right 切換 tab；切換後 focus 自動跳到該分類第一筆。

### 列表欄

- 每行高 32px、font 16px。
- 顯示**只有 title**（不顯示 body 預覽）。
- 240px 寬可塞 ~14 筆可見；超出時列表自動垂直捲動跟隨 focus（Godot ScrollContainer + 自動 ensure_control_visible）。
- 行間 hover / focus 視覺：
  - 預設 title 字色依 status 決定（見下）。
  - 鍵盤 focus 行：2px 橙紅外框 + 行底色微亮。
- 空分類：列表顯示「（尚無筆記）」placeholder，右側全文欄空白。

### `status` 視覺

筆記 `status` 欄位 MVP 渲染：

- `active`：title 字色米白 `(0.94, 0.92, 0.84, 1.0)`。
- 其他值（`superseded` / `inactive` / 未來新增）：title 字色灰 `(0.55, 0.55, 0.55, 1.0)`。

1-C 階段所有預載筆記都是 `active`，此規則為 Phase 2 / 3 預留。

### 全文欄

- 用 `RichTextLabel` 顯示 `body`，`scroll_active = true`。
- 字 16px，行高建議 1.5。
- Body 超出欄高時可垂直捲動：滑鼠滾輪 / Page Up / Page Down / Up / Down（當 focus 在全文區時）。
- 切到不同筆記時全文捲動位置重置回頂。

### 操作 focus 流向

```text
進入筆記 UI    → focus 在「第一個非空分類」的第一筆筆記
Left / Right   → 切 tab，focus 跳到該分類第一筆
Up / Down      → 在列表內上下選筆記；超出列表邊界不繞回
                 列表捲動跟隨 focus（自動 ensure_control_visible）
切到不同筆記   → 全文欄即時更新、捲動回頂
J / Esc        → 關閉筆記
```

全文欄本身**不參與 Tab focus 鏈**，玩家不需要切到全文欄就能捲動 — 在列表 focus 上按 Page Up / Page Down 直接捲動全文（簡化操作）。

資料範例：

```gdscript
{
  "id": "identity_door_unlock_method",
  "category": "身份",
  "title": "門鎖的開法",
  "body": "你想起來了。門鎖需要先按住門把下方的舊感應片, 再用右手手套靠近讀取器。",
  "status": "active"
}
```

### 驗收

- `J` 開啟筆記後 Left / Right 切分類、Up / Down 選筆記，選到時自動顯示全文。
- 同一筆記重複新增不會出現重複條目（以 `id` 去重）。
- 筆記 UI 開啟期間遵守 UI Mode 規則。
- 開啟期間世界仍透出（半透明 overlay）。
- 切到無筆記分類時顯示「（尚無筆記）」placeholder，不 crash。
- panel 尺寸 880×560 置中於 viewport，沿用 inventory skin。
- 選中 tab 顯示底部 2px 橙紅高亮線；未選中 tab 字色暗灰。
- 列表項僅顯示 title；status 非 `active` 的筆記字色灰。
- 列表超過 14 筆時可垂直捲動，鍵盤 focus 在列表內導航時自動 ensure visible。
- 長 body 在全文欄可垂直捲動，切到其他筆記時捲動位置重置回頂。
- INVENTORY 中按 `J` 直接切到筆記（背包關、筆記開），無雙重 panel 疊加。
